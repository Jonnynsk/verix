# This is NOT the Next.js you know

This version has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.
# Глобальные правила для Cursor Agent

## 1. Стек и стилизация
- **Стили:** Всегда используй .module.scss в стиле БЭМ. КАТЕГОРИЧЕСКИ запрещено использовать inline-стили, если об этом не попросили явно.
- **Типизация:** Только TypeScript. Избегай использования `any`, пиши интерфейсы и типы для всех пропсов.
- **Иконки:** Если из Фигмы приходят иконки, старайся использовать SVG напрямую или библиотеку `lucide-react`.

## 2. Архитектура Next.js (App Router)
- По умолчанию все компоненты должны быть **Server Components**. 
- Директиву `"use client"` добавляй ТОЛЬКО в самом верху файла и ТОЛЬКО если компонент использует хуки (`useState`, `useEffect`, `useRef`) или обработчики событий (`onClick`, `onChange`).
- Если нужен интерактивный клиентский элемент внутри серверного компонента, выноси этот элемент в отдельный файл.
- Для изображений всегда используй `<Image />` из `next/image`.

## 3. Генерация из Figma (через MCP)
- При конвертации дизайна обращай пристальное внимание на Auto Layout: переноси его в соответствующие классы `flex`, `grid`, `gap`, `justify-*`, `items-*` из Tailwind.
- Строго соблюдай размеры шрифтов, цвета и скругления (border-radius).
- Разбивай сложные блоки из Фигмы на логические, небольшие подкомпоненты (например, `HeaderLogo.tsx`, `HeaderNav.tsx`).

## 4. Качество кода и язык
- Пиши лаконичный код без лишних оберток `<div>`. Используй фрагменты `<>...</>`, где это возможно.
- Названия переменных и функций — на английском, четкие и понятные.
- Если нужно оставить комментарий, объясняющий сложную логику, пиши его на русском языке.
